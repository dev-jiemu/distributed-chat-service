# 부하테스트 가이드
#### 현재의 docker-compose 환경에서 부하테스트 코드 작성해서 돌려보기
#### feat. Claude

### 1. 대량 메시지 동시 처리
- 10개 스레드에서 각각 1000개 메시지 전송
- 총 10,000개 메시지 처리 성능 측정

### 2. 순간 부하 테스트 (Burst Load)
- 5000개 메시지를 한번에 전송
- 시스템의 순간 처리 능력 측정

### 3. 커넥션 풀 성능
- 20개 동시 커넥션으로 메시지 전송
- 커넥션 관리 효율성 측정

### 4. 속도 제한 테스트
- 초당 100개 메시지로 일정하게 전송
- 시스템의 안정성과 일관성 측정

## 성능 지표

- **처리량(Throughput)**: 초당 처리 가능한 메시지 수
- **레이턴시(Latency)**: 메시지 전송에 걸리는 시간
- **에러율(Error Rate)**: 실패한 메시지 비율
- **리소스 사용량**: CPU, 메모리, 네트워크 사용률

## 튜닝 포인트

1. **RabbitMQ 설정**
   - Connection Pool 크기
   - Prefetch Count
   - Queue 크기 제한

2. **Redis 설정**
   - Connection Pool 크기
   - Timeout 설정
   - 메모리 제한

3. **애플리케이션 설정**
   - Thread Pool 크기
   - 비동기 처리 설정
   - 메시지 배치 처리
